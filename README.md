# New Genesis Bot

Discord-бот с системой тикетов (центром поддержки). Пользователь открывает обращение
через панель с кнопками, бот создаёт приватный канал, где остаются только автор и
администрация. Тикет можно **принять** и **закрыть** — при закрытии формируется лог
переписки и отправляется в ЛС автору и принявшему сотруднику.

## Возможности

- 🎫 Панель тикетов с выбором категории и кнопкой «Открыть тикет».
- 🔒 Приватный канал на каждый тикет (видят только автор, бот и роли поддержки).
- ✅ Кнопки «Принять» и «Закрыть» с подтверждением закрытия.
- 📋 Лог переписки в ЛС при закрытии тикета.
- 💾 Состояние сохраняется в `data/tickets.json` и восстанавливается после перезапуска
  (панель и кнопки продолжают работать).
- 🛡️ Slash- и `!`-команды доступны только участникам с правом **Администратор**.

## Установка

```bash
python -m venv .venv
.venv\Scripts\activate        # Windows
# source .venv/bin/activate   # Linux / macOS

pip install -r requirements.txt
```

## Настройка

1. Скопируйте `.env.example` в `.env`.
2. Укажите `DISCORD_TOKEN` и заполните параметры тикетов:
   - `TICKET_PANEL_CHANNEL_ID` — канал, где будет панель;
   - `TICKET_CATEGORY_ID` — категория для тикетов;
   - `TICKET_STAFF_ROLE_ID` — роли поддержки (через запятую).
3. В настройках бота на Discord Developer Portal включите **Server Members Intent**.

## Запуск

```bash
python main.py
```

После запуска бот опубликует панель тикетов в указанном канале. Панель и кнопки
восстанавливаются автоматически при перезапуске.

## Деплой на Railway

- Добавьте Volume с mount path `/app/data` и переменную `DATA_DIR=/app/data`,
  чтобы тикеты сохранялись между перезапусками.
- Все переменные из `.env.example` задайте в настройках проекта Railway.
